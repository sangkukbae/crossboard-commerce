# 고객사 시연 시나리오

## 1. 운영 인수 가능성 제시

Modern Storefront 첫 화면에서 5개 서비스 구성, 시드 상품 수, 재고 경고를 보여줍니다. 이 화면은 채용 담당자가 전체 범위를 30초 안에 파악하도록 만든 운영 관제형 화면입니다.

## 2. 구매 전환 흐름

상품 카드의 `장바구니 담기`를 누르면 React Query 낙관적 업데이트로 합계가 즉시 바뀝니다. 수량 변경 후 `테스트 결제 승인`을 누르면 주문번호 발급과 결제 확인을 연속 실행합니다.

## 3. 관리자 운영 흐름

`운영 데이터 불러오기`를 눌러 오늘 주문 수, 실제 7일 매출, 재고 부족 상품을 조회합니다. 주문 상태 셀렉트를 변경하면 이력 API가 함께 기록됩니다.

## 4. 1688 도매 연동 흐름

검색어와 카테고리로 도매 상품을 조회하고 CNY 원가, KRW 환산가, 예상 마진을 비교합니다. `판매 상품으로 가져오기`를 누르면 상품 목록에 즉시 추가됩니다.

## 5. ML 추천 운영 흐름

상품 카드의 `유사 상품 보기`를 눌러 API Server → FastAPI 추천 호출과 점수, 캐시 응답 출처를 확인합니다. 운영 환경에서는 Ko-SBERT/FAISS 인덱스 재구축으로 교체할 수 있습니다.

## 6. 모바일·레거시 인수 흐름

Mobile Commerce에서는 키 미설정 시 카카오 데모 로그인을 실행하고 중앙 장바구니를 갱신합니다. Legacy Storefront에서는 상품 상세, 수량 조절, 삭제를 모두 jQuery Ajax로 실행합니다.
